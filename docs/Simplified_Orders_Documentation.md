# 📦 نظام الطلبات المبسط - Quick API
## دليل مطور تطبيقات الموبايل

---

## 📋 نظرة عامة

نظام Quick يدعم نوعين من الطلبات بنظام حالات موحد ومبسط:

| النوع | الوصف | المصدر |
|-------|-------|--------|
| **الطلب العادي (Order)** | منتجات من المتاجر المسجلة | Checkout |
| **الطلب الخاص (CustomOrder)** | اطلب أي شيء - وصف + موقع | إنشاء مباشر |

### ✨ التحديثات الجديدة (v2)
- ✅ **الطلبات الموحدة** - جلب Orders + CustomOrders معاً في endpoint واحد
- ✅ **السائق لا يستطيع الإلغاء** - بعد قبول الطلب، فقط الإدارة تستطيع الإلغاء
- ✅ **شروط أهلية السائق** - رصيد المحفظة + عدد الطلبات النشطة
- ✅ **حقل `order_type`** - يميز نوع الطلب في الاستجابة الموحدة
- 🆕 **إعادة الطلب (Reorder)** - إعادة طلب طلبية مكتملة بالأسعار الحالية

## 🔄 الحالات الأربعة المبسطة

```
    ┌─────────────────────────────────────────────────────────────┐
    │                    الحالات الأربعة                           │
    ├─────────────────────────────────────────────────────────────┤
    │                                                             │
    │   ┌──────────┐      ┌──────────┐      ┌──────────┐         │
    │   │  معلق    │ ───► │قيد التوصيل│ ───► │تم التسليم│         │
    │   │ pending  │      │ shipping │      │delivered │         │
    │   └────┬─────┘      └────┬─────┘      └──────────┘         │
    │        │                 │                                  │
    │        │                 │                                  │
    │        ▼                 ▼                                  │
    │   ┌────────────────────────────┐                           │
    │   │          ملغي               │                           │
    │   │       cancelled             │                           │
    │   │      (مع سبب)               │                           │
    │   └────────────────────────────┘                           │
    │                                                             │
    └─────────────────────────────────────────────────────────────┘
```

| الحالة | القيمة | الوصف بالعربية |
|--------|--------|----------------|
| **معلق** | `pending` | تم الطلب وبانتظار قبول سائق |
| **قيد التوصيل** | `shipping` | السائق قبل ويوصّل الطلب |
| **تم التسليم** | `delivered` | تم التسليم للزبون ✅ |
| **ملغي** | `cancelled` | ملغي أو فشل التسليم (مع سبب) |

## ⏱️ آلية انتظار السائق

```
    ┌─────────────────────────────────────────────────────────────────────┐
    │                     تدفق انتظار السائق                              │
    └─────────────────────────────────────────────────────────────────────┘

    1️⃣ إنشاء الطلب
         │
         ▼
    ┌──────────────────┐
    │    pending       │  ← confirmation_expires_at = +5 دقائق
    │ is_available=true│
    └────────┬─────────┘
             │
       ┌─────┴─────┐
       │           │
       ▼           ▼
    ╔═══════╗   ╔══════════════════╗
    ║ سائق  ║   ║  انتهت الصلاحية   ║
    ║ قبِل  ║   ║ is_confirmation  ║
    ╚═══╤═══╝   ║ _expired = true  ║
        │       ╚═════════╤════════╝
        │                 │
        ▼                 ▼
    ┌────────┐   ┌──────────────────────┐
    │shipping│   │      pending         │
    │        │   │ can_resend = true    │
    └────────┘   │ زر "تأكيد الطلب" يظهر │
                 └──────────┬───────────┘
                            │
                            ▼ المستخدم يضغط /resend
                 ┌──────────────────┐
                 │    pending       │  ← expires_at جديد +5 دقائق
                 │ is_available=true│
                 └──────────────────┘
```

### الملخص:
- عند إنشاء الطلب: `pending` + `confirmation_expires_at = now + 5 دقائق`
- إذا انتهت الصلاحية بدون قبول سائق: **الطلب يبقى pending** ولكن `can_resend_to_drivers = true`
- المستخدم يضغط `/resend` لتجديد الصلاحية وإرسال إشعارات جديدة للسائقين
- **الطلب لا يُلغى تلقائياً** عند انتهاء الصلاحية

### 🔄 إعادة طلب طلبية مكتملة (Reorder)

> يمكن للمستخدم إعادة طلب طلبية مكتملة (تم التوصيل) لإنشاء طلب جديد بالأسعار الحالية.
> - **الأسعار**: يتم استخدام الأسعار الحالية للمنتجات (وليس الأسعار القديمة)
> - **التوفر**: يتم التحقق من توفر المنتجات والكميات
> - **إشعار**: إذا كانت بعض المنتجات غير متوفرة، يتم إنشاء الطلب للمنتجات المتوفرة فقط مع إشعار

**Request:**
```http
POST /api/orders/1/reorder
Authorization: Bearer {user_token}
Content-Type: application/json

{
    "delivery_address": "حماة، حي الصابونية، بناء رقم 15",
    "delivery_fee": 15000,
    "is_immediate_delivery": true,
    "notes": "الرجاء الاتصال قبل الوصول"
}
```

> **ملاحظة:** جميع الحقول اختيارية. إذا لم يتم إرسالها، سيتم استخدام قيم الطلب الأصلي.

## 🚗 شروط أهلية السائق لقبول الطلب

| الشرط | الوصف | رسالة الخطأ |
|-------|-------|-------------|
| **حالة الاتصال** | يجب أن يكون `online` | يجب أن تكون متصلاً لتقبل الطلب |
| **رصيد المحفظة** | يجب أن يكون ≥ نسبة ربح التوصيل | رصيد محفظتك غير كافٍ لقبول هذا الطلب |
| **الطلبات الفورية** | حد أقصى طلب فوري واحد نشط | لديك طلب فوري قيد التوصيل بالفعل |
| **الطلبات المجدولة** | حد أقصى 3 طلبات مجدولة نشطة | لقد وصلت للحد الأقصى من الطلبات المجدولة (3 طلبات) |

```
    ┌─────────────────────────────────────────────────┐
    │           شروط قبول الطلب للسائق                │
    ├─────────────────────────────────────────────────┤
    │                                                 │
    │  💰 رصيد المحفظة ≥ ربح التوصيل؟                 │
    │          │                                      │
    │    ❌ لا │ ✅ نعم                               │
    │          ▼                                      │
    │  📦 طلب فوري أم مجدول؟                         │
    │      │           │                              │
    │   فوري        مجدول                            │
    │      │           │                              │
    │      ▼           ▼                              │
    │  طلبات فورية   طلبات مجدولة                    │
    │  نشطة < 1؟     نشطة < 3؟                       │
    │      │           │                              │
    │   ✅/❌        ✅/❌                             │
    │                                                 │
    └─────────────────────────────────────────────────┘
```

## 🔐 قواعد الإلغاء

### الطلبات الفورية (is_immediate = true)

| من يلغي | في حالة `pending` | في حالة `shipping` |
|---------|-------------------|---------------------|
| **المستخدم** | ✅ مسموح | ❌ غير مسموح |
| **السائق** | ❌ غير مسموح | ❌ غير مسموح |
| **الإدارة** | ✅ مسموح | ✅ مسموح |

### الطلبات المجدولة (is_immediate = false)

| من يلغي | في حالة `pending` | في حالة `shipping` |
|---------|-------------------|---------------------|
| **المستخدم** | ✅ مسموح | ❌ غير مسموح |
| **السائق** | ❌ غير مسموح | ✅ مسموح (مع سبب) |
| **الإدارة** | ✅ مسموح | ✅ مسموح |

### إلغاء السائق للطلب المجدول

عند إلغاء السائق لطلب مجدول، يتم:
1. إلغاء الطلب وتغيير الحالة إلى `cancelled`
2. إرسال البيانات التالية للإدارة:
   - تفاصيل الطلب كاملة
   - تفاصيل المستخدم (الاسم + الهاتف)
   - تفاصيل السائق
   - سبب الإلغاء

**Request:**
```http
POST /api/driver/orders/1/cancel
Authorization: Bearer {driver_token}
Content-Type: application/json

{
    "reason": "العميل غير متواجد في الموعد المحدد"
}
```

**Response:**
```json
{
    "success": true,
    "message": "تم إلغاء الطلب المجدول بنجاح",
    "data": {
        "order": {
            "id": 1,
            "status": "cancelled",
            "statusText": "ملغي"
        },
        "notification_sent_to_admin": true,
        "message_to_driver": "تم إلغاء الطلب وإرسال البيانات للإدارة"
    }
}
```

> ⚠️ الحالات `delivered` و `cancelled` لا يمكن إلغاؤها

## 🔐 Authentication

```
Authorization: Bearer {token}
Accept: application/json
```

- **User APIs**: Guard `api`
- **Driver APIs**: Guard `driver`
- **Admin APIs**: Guard `api` (مستخدم بصلاحيات إدارية)

## ⚠️ Error Handling

| الكود | الرسالة |
|-------|---------|
| `400` | لا يمكن إلغاء هذا الطلب في حالته الحالية |
| `400` | يجب إضافة عنصر واحد على الأقل |
| `400` | تم قبول هذا الطلب من سائق آخر |
| `400` | حسابك غير نشط حالياً |
| `400` | رصيد محفظتك غير كافٍ لقبول هذا الطلب |
| `400` | لديك طلب فوري قيد التوصيل بالفعل |
